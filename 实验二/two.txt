这是我学习的git更新内容。
首先是git clone + ssh的网址，此步骤是远程仓库的内容克隆到本地仓库。
再就是更改本地的内容后上传到远程仓库。git add .用来提交所有的修改文档，这里是提交到暂存区。git commit -m “对本次添加的描述”，以上步骤是从暂存区到本地仓库
再git push操作，从本地仓库到远程仓库。
再git pull操作，从远程仓库到本地仓库。
hi实验报告：准备工作，项目经理干的什么，组员都干的什么
演示：
主程序员任务：
1.创建一个主程序员仓库，公共的，可以在GitHub中创建和编写文件。
   复制网址发给其他程序员，其他人可以通过fork把主程序员仓库下载到自己的GitHub。
2.把仓库克隆到本地，再添加实际项目文件文件，然后从本地上传到github。
  步骤是先git add .提交所有文档，这里是提交到暂存区。git commit -m “对本次添加的描述”，这里是是从暂存区到本地仓库
  然后再git push,从本地仓库上传到主程序员的GitHub。
其他程序员任务：
3.主程序员的工作做完之后，程序员1从主仓库fork到自己的GitHub，然后clone到本地。
  可以再修改提交自己的GitHub，再申请上传到主程序员的仓库，主程序员再同意。
同意之后程序员2.3等需要从主程序员仓库更新到自己GitHub仓库，new pull request.然后在更新到本地（git pull）
4.版本回退，先是git log，显示版本号信息，复制你需要回退到哪一步的版本号。然后git reset --hard 版本号，此步骤是本地回退。然后git push -f强制回退自己GitHub的版本。
5.idea上传到GitHub。
